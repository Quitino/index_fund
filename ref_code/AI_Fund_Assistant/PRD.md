## 1. 项目信息

| 项目 | 内容 |
|------|------|
| Language | 简体中文 |

### 原始需求复述

在现有 daily_stock_analysis 项目旁构建独立的基金量化交易系统。核心流程：交易日定时获取目标基金实时数据 → 有效量化因子分析 → 输出加减仓/不动观望建议 → 飞书/QQ机器人推送。要求基金池、IM推送接口、计划推送时间、有效因子、分析结果报告内容项均支持前台增删改；AI助手可前台开关控制，开启后前端提供对话窗口，支持对基金池全部/个别基金、整个基金或股票市场行情分析。

## 2. 产品定义

### Product Goals

1. **信号驱动决策**：基于加权评分量化因子体系，为基金投资者提供当日加仓/减仓/观望的结构化操作建议，降低主观判断偏差
2. **全链路可配置**：基金池、因子、推送渠道、推送时间、报告内容项均可通过Web前台动态管理，零代码调整运营策略
3. **AI增强洞察**：可开关的AI对话窗口，支持对单只基金、基金池整体、市场行情的智能分析与自然语言问答，提升信息获取效率

### User Stories

| # | 角色 | 用户故事 |
|---|------|---------|
| US1 | 基金投资者 | As a 基金投资者, I want 交易日收到飞书/QQ推送的量化分析建议 so that 我能及时做出加减仓或观望决策 |
| US2 | 基金投资者 | As a 基金投资者, I want 看到每只基金的因子评分明细和加权总分 so that 我能理解建议背后的量化逻辑 |
| US3 | 基金投资者 | As a 基金投资者, I want 通过AI对话窗口询问"今天沪深300ETF适合加仓吗" so that 我能获得个性化的市场洞察 |
| US4 | 系统管理员 | As a 系统管理员, I want 在Web前台增删基金池中的基金代码 so that 我能灵活调整监控范围 |
| US5 | 系统管理员 | As a 系统管理员, I want 在前台配置推送时间、因子权重、报告内容项 so that 无需修改代码即可调整运营策略 |
| US6 | 系统管理员 | As a 系统管理员, I want 一键开关AI助手功能 so that 不使用AI时不会产生额外API开销 |

## 3. 技术规范

### Requirements Pool

#### P0 — Must Have（必须有）

| ID | 需求 | 说明 |
|----|------|------|
| P0-01 | 基金池管理 | Web前台CRUD基金代码（支持ETF/场外基金），含基金名称、代码、类型标签 |
| P0-02 | 量化因子分析引擎 | 加权评分机制：买入信号≥3.5分、卖出信号≥3分。因子含PE百分位、股债性价比（FED模型）、MACD信号、均线趋势、成交量变化等 |
| P0-03 | 分档买入策略 | 信号强度分档：3.5-4.0轻度加仓、4.0-4.5中度加仓、≥4.5重度加仓，对应不同建议仓位比例 |
| P0-04 | 交易日定时调度 | 基于A股交易日历，支持配置调度时间（如14:50收盘前分析、15:30盘后推送），前台可增删改推送时间计划 |
| P0-05 | 飞书推送 | 支持飞书机器人Webhook推送，标题格式：📊A股量化监控日报 YYYY-MM-DD HH:MM，内容含基金名称、信号方向、因子评分、操作建议 |
| P0-06 | QQ机器人推送 | 支持QQ机器人API推送，消息格式与飞书适配 |
| P0-07 | 推送接口前台配置 | Web前台增删改推送渠道（飞书/QQ），配置Webhook地址、推送时间、启用/禁用 |
| P0-08 | 分析报告内容配置 | 前台可配置报告包含的内容项（如：因子明细/加权总分/操作建议/分档建议/风险提示），增删改排序 |
| P0-09 | 因子管理前台配置 | 前台可增删改因子（因子名称、数据源字段、权重、方向正向/反向），调整后即时生效 |
| P0-10 | 数据可视化 | 基金分析结果展示采用红涨绿跌配色（中国A股惯例），含信号灯、评分仪表盘、因子雷达图 |

#### P1 — Should Have（应该有）

| ID | 需求 | 说明 |
|----|------|------|
| P1-01 | AI助手总开关 | 前台开关按钮控制AI功能启用/禁用，关闭时前端隐藏对话入口，后端不调用AI接口 |
| P1-02 | AI对话窗口 | 前端浮动对话窗口，支持自然语言问答，对话历史保留 |
| P1-03 | AI单基金分析 | 对话窗口支持"分析基金510300"等指令，返回单只基金的AI综合分析 |
| P1-04 | AI基金池分析 | 对话窗口支持"分析基金池整体风险"等指令，返回基金池维度的AI分析 |
| P1-05 | AI市场行情分析 | 对话窗口支持"分析今天A股市场行情"等指令，返回大盘/行业板块的AI分析 |
| P1-06 | 数据源适配 | 对接AKShare/TuShare/Baostock等获取基金净值、PE、PB、股债利差等数据 |

#### P2 — Nice to Have（可以有）

### 页面说明

#### P1 - 仪表盘（Dashboard）
- **顶部统计卡**：监控基金数、今日信号数（加仓X/减仓X/观望X）、上次推送时间
- **信号概览表**：今日各基金信号方向（🟢加仓/🔴减仓/⚪观望）、加权评分、操作建议，配色红涨绿跌
- **因子雷达图**：选中某基金后展示各因子得分的雷达图

#### P2 - 基金池管理
- **基金列表**：表格展示基金代码、名称、类型（ETF/场外）、状态（启用/停用）、操作
- **新增/编辑弹窗**：输入基金代码（自动拉取名称）、选择类型、设置标签
- **批量操作**：批量启用/停用/删除

#### P3 - 因子管理
- **因子列表**：表格展示因子名称、数据源字段、权重、方向、状态
- **权重调节**：滑块或数字输入调整权重，实时显示权重占比百分比
- **新增/编辑弹窗**：配置因子参数

#### P4 - 推送配置
- **推送渠道列表**：飞书/QQ等，每项含Webhook地址、启用状态、操作
- **新增渠道弹窗**：选择渠道类型、填写Webhook/Token、测试连接按钮

#### P5 - 报告配置
- **报告内容项列表**：可拖拽排序，每项含名称、启用状态
- **预览区**：实时预览当前配置下的报告样式

#### P6 - 调度计划
- **调度列表**：每条含任务名称、cron表达式/时间点、关联推送渠道、启用状态
- **日历视图**：可视化展示一周内的推送时间节点

#### P7 - AI对话窗口（浮动组件，P1功能）
- **触发方式**：右下角浮动按钮，点击展开对话面板
- **对话界面**：消息列表 + 输入框，支持预设快捷指令（"分析基金池"、"今日市场"）
- **快捷指令按钮**：分析基金池全部 / 分析选中基金 / 市场行情
- **AI开关状态**：顶部栏开关关闭时，对话按钮不显示
